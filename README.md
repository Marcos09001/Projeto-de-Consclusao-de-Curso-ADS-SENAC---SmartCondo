🏢 SmartCondo – Sistema Inteligente para Gestão de Manutenção Condominial

O SmartCondo é um sistema web desenvolvido com o objetivo de modernizar e otimizar a gestão de manutenções em condomínios residenciais. A plataforma centraliza o registro, acompanhamento e gerenciamento de manutenções preventivas e corretivas, proporcionando maior controle, rastreabilidade e organização das atividades realizadas no condomínio. O projeto foi desenvolvido como parte da disciplina de Projeto Integrador, aplicando conceitos de Engenharia de Software, Banco de Dados, UX/UI e Desenvolvimento Web.

📋 Sobre o Projeto

A administração de manutenções prediais ainda é realizada, em muitos casos, por meio de processos manuais, dificultando o controle de históricos, prazos e custos. O SmartCondo foi idealizado para solucionar esse problema por meio de uma plataforma digital que possibilita:

Cadastro de usuários;
Cadastro de condomínios;
Agendamento de manutenções;
Controle de manutenções preventivas e corretivas;
Acompanhamento do status das solicitações;
Exportação de dados em Excel e CSV;
Relatórios gerenciais;
Controle de acesso por autenticação;
Interface responsiva para Desktop e Mobile.

🎯 Objetivos

O projeto tem como principal objetivo oferecer uma solução tecnológica para o gerenciamento de manutenção predial, auxiliando síndicos, administradores e gestores na organização das atividades de manutenção, seguindo as recomendações da ABNT NBR 5674:2024 e boas práticas relacionadas à segurança da informação e à LGPD.

🏗 Arquitetura do Projeto

O SmartCondo foi desenvolvido utilizando uma arquitetura modular, permitindo maior organização do código, facilidade de manutenção e escalabilidade da aplicação.
O desenvolvimento ocorreu de forma incremental, contemplando:

Levantamento de requisitos;
Modelagem de banco de dados;
Protótipos de interface;
Implementação das funcionalidades;
Testes funcionais;
Testes de regressão;
Validação dos requisitos.

🗄 Modelagem do Banco de Dados

O banco de dados foi inicialmente modelado no MySQL Workbench, utilizando o modelo Entidade-Relacionamento (ER), sendo posteriormente implementado em PostgreSQL, garantindo integridade referencial, consistência e escalabilidade dos dados.
